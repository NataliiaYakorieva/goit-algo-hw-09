# Домашнє завдання 9: Жадібні алгоритми та динамічне програмування

## Опис

У цьому завданні реалізовано дві функції для видачі решти:
- `find_coins_greedy` — використовує жадібний алгоритм;
- `find_min_coins` — використовує динамічне програмування для знаходження мінімальної кількості монет.

## Порівняння ефективності

- Жадібний алгоритм працює за O(N), де N — кількість номіналів монет. Він дуже швидкий, оскільки кількість номіналів зазвичай невелика.
- Динамічне програмування працює за O(amount × N), де amount — сума, яку потрібно видати. Для великих сум цей алгоритм працює повільніше, але гарантує мінімальну кількість монет.

## Висновки

- Для стандартного набору монет (наприклад, [50, 25, 10, 5, 2, 1]) жадібний алгоритм завжди дає оптимальний результат і працює значно швидше.
- Для нестандартних наборів монет (наприклад, [1, 3, 4]) жадібний алгоритм може не дати мінімальної кількості монет, тому у таких випадках слід використовувати динамічне програмування.
- У реальних касових апаратах зі стандартними монетами доцільно використовувати жадібний алгоритм для швидкості, але для універсальних рішень — динамічне програмування.
